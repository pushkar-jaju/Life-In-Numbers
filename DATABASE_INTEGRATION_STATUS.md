## Security & RLS (Row Level Security)

### Client-Side (Browser)
- Uses `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- RLS policies enforce permissions
- Users can only read/write their own data
- Public share cards have `is_public=true` flag

### Server-Side (API)
- Uses `SUPABASE_SERVICE_ROLE_KEY`
- Bypasses RLS (admin access)
- Only for backend operations
- Protected by Clerk authentication

### Webhook
- Uses `SUPABASE_SERVICE_ROLE_KEY`
- Signature verified with Clerk webhook secret
- Only processes `user.created` and `user.updated` events

### Common Issues & Fixes

| Issue | Cause | Fix |
|-------|-------|-----|
| 401 Unauthorized | User not authenticated | Ensure signed in, check Clerk integration |
| 500 Database error | Bad query or missing table | Check Supabase schema, verify table names |
| Webhook not syncing | Secret not configured | Add `CLERK_WEBHOOK_SECRET` to `.env.local` |
| Slow queries | Missing indexes | Supabase auto-indexes, but check performance tab |
